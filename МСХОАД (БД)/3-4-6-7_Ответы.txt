Ответы на вопросы по иерархическим данным:

Как осуществляется хранение иерархических данных?
Иерархические данные часто хранятся с помощью деревьев или графов. В реляционных базах данных для этого могут использоваться ссылки на родительские элементы, например, с помощью поля, которое хранит идентификатор родительского элемента в каждой записи (ссылочная модель). Также используются специальные структуры, такие как вложенные таблицы (например, таблицы с рекурсивными связями).

Какие типичные задачи решаются при хранении иерархических данных?

Построение иерархических структур (например, организация каталогов).
Управление родительско-дочерними связями.
Поиск и извлечение данных с учетом иерархии.
Визуализация иерархий (например, дерево каталогов).
Для чего предназначен иерархический тип данных?
Этот тип данных используется для представления объектов, которые имеют явную иерархическую структуру, где каждый элемент (например, запись) связан с одним или несколькими элементами выше (родителями) и/или ниже (дочерними).

Перечислите известные вам свойства и методы иерархического типа данных.

LEVEL — уровень элемента в иерархии.
CONNECT BY — выражение для построения иерархии.
START WITH — начальная точка иерархии.
PRIOR — для указания родительского элемента в иерархии.
Поясните секции иерархических запросов в Oracle.

START WITH — указывает начальную точку для построения иерархии.
CONNECT BY — определяет правила связи между родительскими и дочерними элементами.
ORDER SIBLINGS BY — определяет порядок вывода элементов на одном уровне.
Перечислите известные вам псевдофункции иерархических запросов в Oracle.

LEVEL — возвращает уровень элемента в иерархии.
SYS_CONNECT_BY_PATH — возвращает путь к текущему элементу.
CONNECT_BY_ROOT — возвращает корень текущего пути.
Ответы на вопросы по расширенным группировкам:

Что такое расширенные группировки?
Расширенные группировки позволяют выполнять агрегацию с различными уровнями группировки данных, в том числе использовать несколько уровней иерархии.

Приведите примеры использования расширенных группировок.
Пример использования: можно агрегировать данные сначала по одному набору столбцов, а затем по другому, например, по годам и регионам, и получить сводные результаты.

Для чего предназначена функция GROUPING()?
Функция используется для различения строк с реальными значениями и строк, созданных для группировки.

Для чего предназначена функция GROUP_ID()?
Функция возвращает уникальный идентификатор для каждой группы в агрегированных данных.

Для чего предназначена функция GROUPING_ID()?
Функция используется для возврата уникального значения, которое позволяет идентифицировать комбинации столбцов, по которым происходят группировки.

Что такое составные столбцы?
Составные столбцы — это столбцы, которые создаются как комбинация нескольких отдельных столбцов в запросах.

Что такое GROUPING SETS?
GROUPING SETS позволяют выполнять несколько агрегаций по разным комбинациям столбцов в одном запросе.

Поясните синтаксис аналитических функций.
Синтаксис: function_name (expression) OVER (PARTITION BY column ORDER BY column), где PARTITION BY разделяет данные на группы, а ORDER BY — задает порядок внутри группы.

Перечислите виды аналитических функций.

Окна агрегатных функций (например, SUM, AVG).
Ранжирование (например, RANK, DENSE_RANK).
Функции для сдвига строк (например, LEAD, LAG).
Перечислите известные вам аналитические функции каждого вида.

Окна агрегатных функций: SUM(), AVG(), COUNT().
Ранжирование: RANK(), DENSE_RANK(), ROW_NUMBER().
Функции для сдвига строк: LEAD(), LAG().
Поясните секцию ORDER BY аналитических функций.
Эта секция определяет порядок строк в пределах каждой группы или окна. Это важно для таких функций, как ROW_NUMBER() или RANK(), которые зависят от порядка.

Поясните секцию PARTITION BY аналитических функций.
Эта секция делит данные на группы, внутри которых происходит выполнение аналитической функции.

Поясните ключевые слова ROWS и RANGE аналитических функций.

ROWS — задает диапазон строк, которые будут включены в расчет (например, на определенный ряд вперед или назад).
RANGE — включает все строки с одинаковыми значениями, учитывая сортировку.
Опишите, как работает вложенность аналитических функций.
Вложенные аналитические функции позволяют использовать результат одной аналитической функции в другой. Однако при этом нужно соблюдать осторожность, чтобы не перегрузить запрос излишними вычислениями.

Ответы на вопросы по SQLite:

Что такое SQLite?
SQLite — это легковесная, встроенная реляционная база данных, которая не требует отдельного сервера. Она часто используется в мобильных приложениях и встраиваемых системах.

Что такое классы хранения? Какие классы хранения существуют в SQLite?
Классы хранения в SQLite определяют, как данные будут храниться в таблице. В SQLite существует несколько классов, включая INTEGER, REAL, TEXT, и BLOB.

Что такое аффинированные типы? Какие аффинированные типы существуют в SQLite?
Аффинированные типы — это типы данных, которые могут быть не строго привязаны к одному физическому типу, а могут быть приведены к другому типу данных. Пример аффинированного типа в SQLite — это TEXT, который может хранить строки, но может быть также приведен к числовому типу.

Как хранить дату в SQLite?
Даты можно хранить как строку в формате YYYY-MM-DD, как целое число (количество секунд с 1970 года) или как реальное число (количество дней).

Поясните правила сравнения данных в SQLite.
SQLite выполняет сравнение данных с учетом их типов. Для чисел и строк используется стандартное лексикографическое или арифметическое сравнение.

Перечислите объекты SQLite.
Таблицы, индексы, представления, триггеры.

Поясните порядок использования внешних ключей в SQLite.
Внешние ключи в SQLite активируются с помощью PRAGMA foreign_keys = ON;. Они обеспечивают ссылочную целостность между таблицами.

Что такое PRAGMA?
PRAGMA — это инструкция в SQLite, которая используется для выполнения команд, таких как настройка параметров базы данных или получение информации о базе данных.

Какие типы индексов существуют в SQLite?
В SQLite существуют стандартные индексы и уникальные индексы. Стандартные индексы ускоряют поиск, уникальные — обеспечивают уникальность значений.

Какие программные объекты SQLite вы знаете?
Таблицы, индексы, представления, триггеры, а также функции и процедуры, доступные через SQL запросы.

Ответы на вопросы по конструкции MODEL в SQL:

Для чего используется конструкция MODEL?
Конструкция MODEL используется для выполнения вычислений и манипуляций с данными в таблицах, как если бы они представляли собой массивы, что позволяет создавать более сложные и динамичные запросы.

Поясните секцию ORDER BY конструкции MODEL.
Секция ORDER BY определяет порядок обработки строк, прежде чем будет выполнено моделирование или вычисление значений.

Поясните секцию PARTITION BY конструкции MODEL.
Секция PARTITION BY делит данные на группы, по которым выполняется моделирование. Это позволяет выполнять вычисления в пределах каждой группы.

Поясните секцию DIMENSIONS конструкции MODEL.
Секция DIMENSIONS определяет размеры модели, по которым будут осуществляться вычисления.

Что такое мера конструкции MODEL?
Мера — это выражение или столбец, по которому будут выполняться вычисления или агрегации.

Что такое RULES?
RULES — это правила, которые определяют, как будут вычисляться значения в модели.

Что такое символьная, позиционная и смешанная нотации?
Это способы ссылки на ячейки в модели. Символьная нотация использует именованные переменные, позиционная — индексы, а смешанная комбинирует оба подхода.

Что такое NESTED REFERENCES?
Это ссылки на другие вычисления внутри модели, позволяя создавать более сложные и многослойные вычисления.

Что такое REFERENCE MODEL?
Это модель, которая используется как ссылка для других вычислений или моделей.

Для чего используется UNIQUE SINGLE REFERENCE?
Для создания ссылок на уникальные значения в других частях модели.

Для чего используется конструкция MATCH_RECOGNIZE?
Она используется для поиска шаблонов в последовательности строк, что полезно в аналитических запросах.

Что такое переменные шаблона в конструкции MATCH_RECOGNIZE?
Это переменные, которые используются для обозначения частей шаблона, например, событий или состояний.

Что такое шаблон в конструкции MATCH_RECOGNIZE?
Шаблон — это условие, которое определяет последовательность событий или состояний в данных.

Для чего используются меры в конструкции MATCH_RECOGNIZE?
Меры позволяют вычислять значения на основе найденных шаблонов.

Для чего используется ключевая фраза AFTER MATCH в конструкции MATCH_RECOGNIZE?
Эта фраза определяет действия, которые выполняются после того, как шаблон был найден в данных.
